Examples of the jquery.countoverdue timer

So you need an easy way for your webapp to display to the user a timer that counts down to zero; once it gets to zero it needs to continue into the negative and change visually to allow your user an easy cue that things have gone on Far Too Long.

To fill this unique niche, comes jquery.countoverdue!

These examples will show you how jquery.countoverdue works, and with some degree of luck, provide a solution for you quickly.

Default settings

This timer is using the default options of countoverdue. First we add a div that will be the timer.

code:
html:

<div id="blarg1"></div>

Now in our script we create the timer using one of the dates above. Since we are using the defaults, only the date needs to be supplied to countoverdue. You can pass an actual Date() or a date string/number.

js:

$("#blarg1").countoverdue(soon_date);

result:

This second timer uses the time that has already past. The starts in the negative and stays that way.

code:
html:

<div id="blarg2"></div>

js:

$("#blarg2").countoverdue(past_date);

result:

The countoverdue timer can be cancelled by passing "end" to the same div.

js code:

$("#blarg1").countoverdue("end");
$("#blarg2").countoverdue("end");

Here we'll create a button that toggles a countoverdue timer on and off.

code:
html:

<div id="blarg3toggle" class="button">toggle blarg3</div>
<div id="blarg3" class="inline"></div>

js: $("#blarg3").countoverdue(soon_date);
var timer_on = 1;
$("#blarg3toggle").click(function() {
  if (timer_on == 1) {
    $("#blarg3").countoverdue("end");
    timer_on = 0;
  }
  else {
    $("#blarg3").countoverdue(soon_date);
    timer_on = 1;
  }
});

result:

toggle blarg3

This toggle has the appearance of show/hide but it is in fact not. Since countoverdue bases timer value on the supplied date as compared to the current date, ending and starting the same timer looks like it remained running while not visible. This is not the case.

Basic Options and Styling

Countoverdue creates a span with two default classes:

"countoverdue-timer"
"countoverdue-negtimer"

If you use the defaults the "countoverdue-timer" class styles the timer while in the positive, "countoverdue-negtimer" once it becomes overdue and counts into the negative.

In this example, the default classes are styled in the css file "countoverdue-example.css". The entries look like this:

css code:

.countoverdue-timer {
  color : #000000;
}

.countoverdue-negtimer {
  color: #ff0000;
}

As you can see, only the text color is edited.

Using css for these two default classes is the simpliest way to customize countoverdue. Styling the div supplied to countoverdue works also, but the timer will have the same styling in both positive and negative, which may be desired outcome.

Should decide you'd like more control over the timers (perhaps using countoverdue for more than one type of timer on the same page) you can pass countoverdue an object of key:values to adjust things to your liking.

The default option values are:

js code:

var defaults = {
  timer_span : "countoverdue-timer",
  negtimer_span : "countoverdue-negtimer",
  timer_pre : "",
  negtimer_pre : "-",
  separator : ":",
  negseparator : ":",
  timer_post : "",
  negtimer_post : "",
};

Should you supply an object replacing default values, you need to also supply the date as a key:value like so:

code:

$("#my_countoverdue_timer").countoverdue({
  date : "my supplied date",
  timer_span : "my_positive_class",
  negtimer_span : "my_negative_class",
});

The keys "timer_span" and "negtimer_span" are the names of the span classes that countoverdue will use. Here is an example of suppling different class names for the spans:

code:
css:

.countoverdue-timerBLUE {
  color: #0000ff;
}
.countoverdue-negtimerBOLD {
  color: #ff0000;
  font-weight: bold;
}

js:

$("#blarg4").countoverdue({
  date : soon_date,
  timer_span : "countoverdue-timerBLUE",
  negtimer_span : "countoverdue-negtimerBOLD"
});

result:

Oh look! There we have a new timer, styled differently by declaring a different class for both stages of the timer.

Here is a countoverdue timer that uses a non-default class on the negative side (and you thought blink tags were dead, just wait.).

code:
css:

.countoverdue-negtimerBLINK {
  color: #ff0000;
  animation: blink 0.5s step-end infinite;
}

@keyframes blink {
  0% {color: #ff0000}
  75% {color: #ffffff}
}

js:

$("#blarg5").countoverdue({
  date : soon_date,
  negtimer_span : "countoverdue-negtimerBLINK"
});

Serious Style Control & Extra Options

There are six extra options that can be passed to countoverdue.

js code:

var extra_options = {
  timer_pre : "",
  negtimer_pre : "-",
  separator : ":",
  negseparator : ":",
  timer_post : "",
  negtimer_post : "",
};

The two are tell countoverdue what string you want put in front of the timer. You can set different strings for positive and negative timers.

Separator strings are what will appear between the minutes and seconds displayed in the timer.

The post strings are what follows the timer. The default is for the post strings to be empty.

js code:

$("#my_countoverdue_timer").countoverdue({
  timer_pre : "Due in: ",
  negtimer_pre : "Overdue by: ",
  separator : " minutes & ",
  negseparator : " minutes & ",
  timer_post : " seconds.",
  negtimer_post : " seconds!"
});

result:

If you'd like to do some seriously weird things to the style of your countoverdue timer, within the main span of the timer countoverdue creates child spans of the following classes:

  • countoverdue-pretimer, class of the pre-timer string.
  • countoverdue-mins, class of the minutes.
  • countoverdue-separator, class of the separator string.
  • countoverdue-tsecs, class for the seconds in the tens column.
  • countoverdue-osecs, class for the seconds in the ones column.
  • countoverdue-posttimer, class of the post-timer string.
Here is one final example. We'll use all of the options and then just go nuts with the css.
